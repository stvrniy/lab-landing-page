## 2. Стратегія гілкування: GitHub Flow

### Обрана стратегія: **GitHub Flow**

GitHub Flow — це проста та легковага стратегія гілкування, яка ідеально підходить для веб-проектів з безперервним розгортанням.

### Основні принципи:

1. **Гілка `main`** — завжди готова до розгортання, містить стабільний код
2. **Feature-гілки** — створюються для нових функцій з гілки `main`
3. **Pull Requests** — обов'язковий код-рев'ю перед злиттям
4. **Негайне розгортання** — після злиття в `main` код можна відразу деплоїти

### Чому обрано GitHub Flow:

- Простота і зрозумілість для однорічного проекту
- Ідеально для статичних веб-сайтів (HTML/CSS/JS)
- Безпосередня інтеграція з GitHub Pages
- Легка підтримка документації

## 9. Стратегія гілкування — детальний опис

### GitHub Flow — короткий опис

GitHub Flow — це спрощена стратегія гілкування, запропонована GitHub, яка складається з таких правил:

1. **Гілка `main`** — головна гілка, код у ній завжди готовий до розгортання
2. **Робота в окремих гілках** — для кожної нової функції або виправлення створюється окрема гілка
3. **Pull Requests** — зміни обговорюються перед злиттям
4. **Негативне розгортання** — після злиття код можна одразу деплоїти

### Переваги для цього проекту:

- **Простота** — один головний branch без складних правил
- **Ідеально для статичних сайтів** — GitHub Pages деплоїть автоматично з `main`
- **Легкість документування** — теги показують етапи розробки
- **Прозорість** — кожен етап має свій коміт та опис
